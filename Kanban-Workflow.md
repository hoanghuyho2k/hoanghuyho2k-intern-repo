
# Agile Workflows & Kanban

## How a Kanban Board Works

A Kanban board is a visual tool used to manage workflow by organizing tasks into different stages. It helps track progress, identify bottlenecks, and ensure work is completed efficiently.

## Kanban Columns

Typical columns include:
- **Backlog / Not Started**: Tasks that are planned but not yet started
- **In Progress**: Tasks currently being worked on
- **Blocked**: Tasks that cannot proceed due to issues
- **Ready for Review**: Tasks completed and waiting for review
- **Done / Approved**: Tasks that are fully completed and approved

## Task Movement

Tasks move from left to right across the board:
- Developers move tasks when they start working on them
- Tasks are updated as progress is made
- Tasks are moved to review once completed
- Final approval marks the task as done

Each team member is responsible for updating the status of their own tasks to keep the board accurate.

## Work in Progress (WIP) Limits

Limiting WIP means restricting the number of tasks in progress at one time. This helps:
- prevent overload
- improve focus and productivity
- reduce task switching
- identify bottlenecks early

## Reflection

### How Kanban helps manage priorities

Kanban helps manage priorities by:
- clearly showing which tasks need attention
- preventing too many tasks from being started at once
- helping focus on completing tasks before starting new ones

This reduces overload and improves efficiency.

### Improving my workflow

I can improve my workflow by:
- focusing on one task at a time
- updating task status regularly
- avoiding starting new tasks before finishing current ones
