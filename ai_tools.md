# AI Tools for Development

## Which AI tools did you try?

During this onboarding process, I mainly used ChatGPT as an AI assistant to support my development workflow. I used it to help understand Git commands, resolve merge conflicts, and troubleshoot issues when pushing changes to GitHub.

I also researched GitHub Copilot, which is an AI coding assistant integrated directly into Visual Studio Code. Copilot can suggest code as developers type and help speed up development. Although I did not fully rely on Copilot during this task, I explored how it works and how it can assist developers when writing code.

I also looked into Claude AI as another AI assistant that can help with programming questions and explanations.

---

## What worked well? What didn’t?

One thing that worked well was using AI to explain technical problems clearly. For example, when I encountered Git errors such as authentication problems or merge conflicts, the AI helped break down the error messages and guide me step-by-step toward a solution.

AI was also useful for:

- explaining Git concepts like rebase, merge conflicts, and branch management
- helping generate example commands
- clarifying development workflows

However, AI is not perfect. Sometimes the instructions need to be adjusted based on the actual project setup, and it still requires understanding from the developer to apply the suggestions correctly.

Because of this, I found that AI works best when used as a learning assistant, not as a replacement for understanding the code or the tools being used.

---

## When do you think AI is most useful for coding?

I think AI is most useful in the following situations:

- Learning new technologies – AI can explain concepts, frameworks, or tools quickly.
- Debugging errors – AI can help interpret error messages and suggest possible solutions.
- Generating simple code snippets – AI can help speed up writing repetitive or boilerplate code.
- Understanding unfamiliar code – AI can help explain what certain parts of code are doing.

However, developers should still review and understand the code themselves because AI-generated suggestions may not always be correct or optimized.

---

## Conclusion

Overall, AI tools can significantly improve developer productivity when used correctly. They are especially helpful for learning, debugging, and speeding up development tasks. However, it is still important for developers to understand the underlying concepts and verify AI-generated solutions rather than relying on them completely.
