# Learning AI Coding Tools (Cursor)

## AI Tool Used

For this task, I used Cursor, an AI-powered code editor that integrates AI assistance directly into the development environment. Cursor allows developers to ask questions about their code, generate code suggestions, refactor existing code, and explain complex logic within the editor.

---

## What I Used It For

I used Cursor for the following tasks during the onboarding process:

1. **Understanding existing code**

I used Cursor to help understand parts of the repository structure and files in the project. By asking questions about the code and project setup, Cursor helped explain what certain files and commands were doing.

1. **Debugging and explaining Git issues**

While working with Git commands such as merging branches, resolving conflicts, and fixing push errors, I used Cursor to help explain error messages and understand what the correct commands should be.

1. **Writing documentation**

Cursor also helped generate explanations and documentation for some of the markdown files required in the onboarding tasks. It was useful for drafting structured explanations that I could then review and adjust.

---

## What It Helped With

Cursor was particularly helpful for:

- Explaining Git commands and error messages\*\*
- Understanding the structure of the project repository
- Generating documentation drafts
- Speeding up repetitive writing tasks

Because the AI is integrated into the code editor, it is convenient to ask questions directly while working with the code.

---

## What It Struggled With

Although Cursor was helpful, there were some limitations.

Sometimes the explanations needed to be adjusted depending on the exact situation in the repository. AI suggestions can be helpful, but they are not always completely accurate for every project setup.

Because of this, it is important to review the suggestions carefully and understand what the commands or code changes actually do before applying them.

---

## Conclusion

Overall, Cursor is a useful AI-assisted development tool that can help developers understand code, debug issues, and improve productivity. However, it should be used as a support tool rather than a replacement for understanding the code or development process.

Using AI responsibly means reviewing suggestions, verifying commands, and making sure the developer understands the solution rather than relying on the AI completely.
