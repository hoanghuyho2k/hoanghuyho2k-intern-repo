
# AI Usage Guidelines – Reflection

## AI Tools in My Role

As a backend developer intern, I use AI tools such as ChatGPT to:
- understand new concepts (e.g., NestJS, TypeORM)
- debug errors and identify solutions
- generate code examples and improve existing code
- assist with documentation and reflections

## Benefits of Using AI

- improves productivity and speeds up problem-solving
- provides quick explanations and examples
- helps learn new technologies faster
- supports debugging and troubleshooting

## Risks of Using AI

- potential exposure of sensitive or confidential data
- incorrect or outdated information
- over-reliance without understanding the solution
- blindly trusting generated code without verification

## Sensitive Information to Avoid

I should never input:
- API keys, passwords, or tokens
- database credentials
- user personal data
- company internal code or confidential project details
- any sensitive or private information

## Validating AI Output

To ensure accuracy, I will:
- test all generated code before using it
- compare answers with official documentation
- verify logic and correctness
- make necessary adjustments instead of copying directly

## Reflection

### When to use AI vs. own skills

I will use AI when:
- I need guidance, examples, or explanations
- I am stuck on a problem and need direction

I will rely on my own skills when:
- implementing core logic
- making design decisions
- ensuring code quality and correctness

### Avoiding over-reliance

To avoid over-reliance on AI, I will:
- try solving problems myself first
- use AI as a support tool, not a replacement
- understand solutions before applying them

### Ensuring data privacy

To maintain data privacy, I will:
- avoid sharing sensitive or confidential information
- use placeholders instead of real data
- follow company policies when using AI tools

## Task

### AI-assisted task

I used AI to help generate unit tests and API testing code for my NestJS project. This improved my understanding of testing structure and saved time.

### Review of AI output

The AI-generated code required adjustments to match my project structure and dependencies. I reviewed and tested the code to ensure it worked correctly.

### Best practice

One key best practice I will follow is:
- always review, test, and understand AI-generated output before using it in my work

## Key Learning

AI is a powerful tool for learning and productivity, but it must be used responsibly. Critical thinking and data privacy are essential when integrating AI into professional work.
