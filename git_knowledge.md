
# Git Setup & Knowledge Reflection

## Have you used Git before? If so, in what context?

Yes, I have used Git before during my university studies and personal programming projects. I used Git mainly for version control when working on coding assignments and small software projects. Git helped me keep track of changes in my code and allowed me to revert to previous versions if something went wrong.

During this internship onboarding process, I used Git more actively to manage branches, resolve merge conflicts, and push my work to GitHub. This experience helped me better understand how Git is used in a real development workflow.

---

## Which Git client (if any) did you choose? Why?

I mainly used the Git command line interface (CLI) in the terminal. I chose this because it provides full control over Git operations and helps me better understand what each command is doing.

I also used GitHub Desktop occasionally to visualize changes and commits more easily. However, I prefer the command line for most operations because it is faster and commonly used in professional development environments.

---

## What was the most interesting thing you learned about Git today?

One of the most interesting things I learned was how Git handles merge conflicts and rebasing. When working on the repository, I encountered situations where my local changes conflicted with changes from the remote repository.

Through this experience, I learned how to resolve conflicts manually and how commands like `git rebase`, `git merge`, and `git status` help manage changes between branches.

Understanding how Git manages commit history and resolves conflicts gave me a better appreciation of how developers collaborate on the same codebase without losing work.

---

## Reflection

Overall, using Git during this onboarding process helped me become more comfortable with version control in a real development workflow. I learned how to manage branches, resolve conflicts, and push changes to a shared repository. These are important skills for collaborating effectively in a software development team.
