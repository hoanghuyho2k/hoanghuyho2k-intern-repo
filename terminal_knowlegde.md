# Terminal Client Setup & Command Line Reflection

## Which terminal client did you choose? Why?

I used the default macOS Terminal with the zsh shell, which is already installed on Mac. I chose this because it is simple, reliable, and already integrated into the operating system, so there was no need to install additional software.

Using the default terminal also helped me focus on learning the core command line tools and Git commands without needing to configure a new environment.

---

## What customizations (if any) did you make?

I kept most of the terminal settings at their default values. The only customization I made was changing the Profile color theme to "Pro", which uses a black background.

I chose this theme because a darker background is easier on the eyes when working in the terminal for long periods of time.

Other than that, I did not install plugins or create aliases yet because I want to first become more comfortable with the basic commands.

---

## What was the most useful command you learned today?

One of the most useful commands I used today was:
git status

This command shows the current state of the repository, including:
- which branch I am on
- whether my branch is up to date with the remote repository
- which files have been modified or staged
- whether there are untracked files

This command is extremely useful because it quickly tells me what is happening in my Git repository and what actions I need to take next.

---

## Reflection

Working in the terminal helped me better understand how Git works and how development workflows operate behind the scenes. Instead of relying only on graphical tools, using the command line gives more control and a clearer understanding of what is happening when running Git commands.
