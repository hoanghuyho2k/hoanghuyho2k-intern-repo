
# Roles in an Agile Team

## Key Roles in an Agile Team

### Developers
Developers are responsible for building features, fixing bugs, and implementing business logic. They write, test, and maintain code.

### QA (Quality Assurance)
QA ensures the quality of the product by testing features, identifying bugs, and verifying that requirements are met.

### Product Manager (PM)
The Product Manager defines product requirements, prioritizes tasks, and ensures the product delivers value to users.

### Agile Project Manager (Agile PM)
The Agile PM facilitates the Agile process, ensures smooth workflow, removes blockers, and helps the team stay aligned.

### Other Supporting Roles
- **UX Designers**: design user-friendly interfaces and experiences
- **Stakeholders**: provide business direction and requirements
- **Customer Support**: gather user feedback and report issues

## Collaboration in Kanban vs Scrum

- **Scrum**:
  - structured roles and ceremonies
  - fixed sprint cycles
  - clear planning and review processes

- **Kanban**:
  - flexible workflow
  - continuous task prioritization
  - collaboration happens continuously rather than in fixed cycles

## Reflection

### My Role

My role is a **Backend Developer Intern**. I contribute by:
- building APIs using NestJS
- implementing features and backend logic
- debugging and testing code
- supporting the overall development process

### Contribution to Agile Process

I support the Agile process by:
- completing assigned tasks on time
- updating task status on the Kanban board
- communicating progress and blockers
- writing clean and testable code

### Collaboration with Other Roles

I frequently collaborate with the **Product Manager (PM)**.

- The PM provides requirements and priorities
- I translate those requirements into backend implementation
- Clear communication helps ensure the feature meets expectations

### Improving Teamwork

To improve collaboration with the PM, I can:
- ask clarifying questions early when requirements are unclear
- confirm expected outcomes before implementation
- provide regular updates on progress

## Key Learning

Each role in an Agile team has a specific responsibility, but collaboration is essential. Clear communication and understanding of roles help ensure successful project delivery.
